目前的 myNTU 首頁資訊量非常龐大，雖然功能齊全，但缺乏視覺層級與重點。以下是我的重新設計與版面安排提案：

### 1. 視覺風格與 UI 基礎 (Visual & UI Foundation)

* **色彩計畫 (Color Palette)：** 捨棄過多的強調色，採用**「臺大經典色（傅鐘紅/磚紅）與金色」**作為點綴色（Accent Colors），主視覺以大量的**留白（Whitespace）**或淺灰底色為主，降低視覺疲勞。
* **深色模式 (Dark Mode)：** 現代化網頁必備。提供切換開關，深色模式下以深灰/藏青色為主底色，配上低飽和度的紅色點綴，適合學生夜間使用。
* **字體排印 (Typography)：** 統一使用現代的無襯線字體（如 Noto Sans TC 或 PingFang TC），加大標題字號，加強字體粗細對比，讓資訊層級（Hierarchy）一目了然。

### 2. 版面架構重新安排 (Layout Architecture)

我會將版面分為「登入前」與「登入後」兩種完全不同的情境：

#### 【登入前：極簡門面】

* **全螢幕/大版面主視覺：** 採用高畫質且經過適度暗化或毛玻璃模糊處理的臺大校園風景（如總圖、椰林大道），給人寧靜、大氣的感覺。
* **置中登入框 (SSO Card)：** 畫面正中央放置一個具備現代感（帶有輕微陰影、毛玻璃效果或純白圓角）的登入卡片，僅保留帳號、密碼與「SSO 登入」按鈕。
* **重要公告推播：** 在登入框下方或側邊，以輪播或簡潔的跑馬燈形式，僅顯示「全校性緊急公告」或「系統維護通知」，其餘資訊全部隱藏。

#### 【登入後：個人化儀表板 (Dashboard)】

登入後，版面應分為四大區塊：**頂部導航、個人動態總覽、自訂捷徑、服務模組分類**。

**A. 頂部導航列 (Top Navigation Bar - 懸浮且常駐)**

* **左側：** 簡約的 myNTU Logo。
* **中間 (核心功能)：** **全局搜尋列 (Universal Search Bar)**。這是現代化的靈魂，類似 Mac 的 Spotlight。學生不用再去各分類找連結，直接輸入「成績」、「選課」或「請假」，就能跳出直達連結。
* **右側：** 語系切換 (EN/中)、通知鈴鐺（紅點提示新訊息）、使用者頭像與設定。

**B. 歡迎與動態總覽區 (Hero Section)**

* **個人化問候：** 「早安，[姓名]同學 / 教授」。
* **動態資訊卡片 (Smart Widgets)：** 抓取系統資料，主動提示**當下最需要的資訊**。例如：
* *「下一節課：10:20 資料結構 (資工館 101)」*
* *「圖書館：有 2 本書即將逾期」*
* *「繳費：學雜費繳交期限剩餘 3 天」*



**C. 我的最愛捷徑區 (Quick Access Favorites)**

* 取代目前的「十大熱門服務」，改為 **App 圖示化的網格排列 (Grid View)**。
* 系統預設給予最常用的功能（如：NTU COOL、臺大課程網、成績查詢、信箱）。
* **支援客製化：** 使用者可以自由「拖曳排序」或「新增/移除」這些捷徑磚，打造專屬自己的首頁。

**D. 服務目錄 (Service Catalog)**

* 目前的分類（教職申辦、學生專區、圖書研究等）過於龐雜。我會採用**「漸進式揭露 (Progressive Disclosure)」**的設計。
* 將功能收納進幾個大分類的**「折疊卡片」**或**「側邊欄 (Sidebar)」**中。例如：【學習與課程】、【生活與校園】、【財務與行政】、【資訊與圖書】。
* **身分識別過濾 (Role-based filtering)：** 如果登入者是學生，系統自動隱藏「教職員薪資」、「公文系統」等無關選項，讓介面保持極致乾淨。

### 3. 其他現代化體驗細節 (UX Enhancements)

* **RWD 響應式設計 (Mobile-First)：** 大學生極度依賴手機。新版設計必須在手機上完美呈現，捷徑區在手機上應變成容易單手點擊的大按鈕，選單則收納進漢堡選單 (Hamburger Menu) 中。
* **微互動動畫 (Micro-interactions)：** 滑鼠懸停在按鈕或卡片上時，加入輕微的上浮效果與陰影變化；點擊切換頁籤時有平滑的過場動畫，提升網站的「精緻感」。
* **整合最新消息佈告欄：** 將原本雜亂的「活動報名」、「校園公佈欄」整合為一個乾淨的頁籤式模組 (Tabbed Widget)，並加上標籤（如 `[演講]`、`[獎學金]`、`[徵才]`），讓資訊更容易掃描。

**總結來說：**
重新設計的 myNTU 不該再是一個「把所有連結攤在桌上」的佈告欄，而應該轉型為一個**「懂使用者的數位助理」**。透過強大的搜尋功能、自動過濾無關資訊、以及高度客製化的常用捷徑，就能在視覺上達到極簡，在操作上達到最高效率。

### 作法一：寫在不同的 HTML 檔案（傳統多頁式應用 MPA）

這也是目前多數老系統（包含現在的 myNTU）最常見的做法。你會擁有兩個實體的檔案，例如 `login.html` 和 `dashboard.html`。

* **運作方式：** 使用者在 `login.html` 輸入帳密，點擊登入後，瀏覽器會將資料送到伺服器驗證。驗證成功後，伺服器會將網頁**「重新導向 (Redirect)」**到 `dashboard.html`。
* **優點：** 權責劃分非常清楚，登入前的頁面不需要載入登入後才用得到的複雜程式碼，安全性控管直觀。
* **缺點：** 登入瞬間，瀏覽器必須重新載入整個新網頁，畫面會「白一下」或有明顯的跳轉感，這在現代化設計中會稍微破壞沉浸式體驗。

### 作法二：寫在同一個 HTML 檔案（現代單頁式應用 SPA）

如果你使用現代前端框架（如 React、Vue.js 或 Angular），**整個網站實際上只有一個 `index.html` 檔案**。

* **運作方式：** 網頁一開始就載入一個核心程式。這個程式會檢查使用者的「登入狀態 (State)」。
* 如果 `isLoggedIn === false`，畫面就畫出【極簡門面】元件 (Component)。
* 如果驗證成功 `isLoggedIn === true`，程式會**直接在原畫面把登入框抽換掉，動態渲染出【個人化儀表板】元件**。


* **優點：** 體驗極致流暢！登入過程像解鎖手機一樣自然，沒有網頁重新載入的頓挫感，還能加入平滑的過場動畫（例如登入卡片淡出、儀表板淡入）。
* **缺點：** 需要學習並導入現代前端框架，技術門檻較高；且首頁一開始就要載入較多 JavaScript。
